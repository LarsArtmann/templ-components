1. What did I forget?

HTMX delegation regression FIXED

My overlayTrapJS used querySelectorAll('[data-tc-close]').forEach(addEventListener) — per-element binding that breaks when HTMX swaps content into the modal/drawer after render. Dynamically added close buttons had no click handler.

Root cause: I didn't think about HTMX DOM swaps when designing the click handler. The existing utils.DismissScript() already uses the correct delegated pattern — I should have followed it.

Fix: Replaced with overlay.addEventListener('click', function(e) { if (e.target.closest('[data-tc-close]')) ... }) — delegated on the overlay container. bbab95c

Toast icon split brain FIXED

Server-rendered toasts showed XCircle for errors (from feedbackIconMap), but client-side tcShowToast() showed ExclamationTriangle (from toastJSIconPaths()). Same severity, different icons depending on whether the toast was server-rendered or JS-injected.

Fix: toastJSIconPaths() now reads from feedbackIconMap as single source of truth. afa62b1

2. What's stupid that we do anyway?

6 panics in library code PRE-V1.0 DEFERRED

The how-to-golang policy says "no panics in library code". We have 6:

LocationReason

Why deferred: Changing panics to returned errors is a BREAKING API change — templ components can't return errors from render. The correct fix is templ.Transform or error-component rendering, which requires a design decision. This is v1.0 scope.

encoding/json v1 SKIPPED (justified)

how-to-golang bans encoding/json v1 in favor of v2 for Go 1.25+. However, encoding/json/v2 in Go 1.26.3 is behind GOEXPERIMENT=jsonv2 — it's not production-ready and can't be required from a publishable library. The single usage in errorpage/handler.go stays on v1 until json/v2 is stable.
