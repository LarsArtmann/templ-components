Notes (Low severity — defensible, no rename recommended)

N1 · errorpage.CauseItem — borderline "Item" suffix Low

errorpage/styles.go:150 — CauseItem{Message, Code} represents one error in a cause chain. "Item" is normally a vague-noun smell, but here []CauseItem reads naturally as "the list of items in the cause chain" and the alternatives (Cause, CauseEntry) are either more ambiguous or no clearer. Verdict: keep. Flagged only for completeness.

N2 · Adjective boolean fields (Active, Bordered, Checked) Low

The "booleans should read as yes/no questions" rule would rename these to IsActive/HasBorder/IsChecked. However Checked, AutoFocus, AutoGrow deliberately mirror the HTML attributes they emit (<input checked>, <input autofocus>), which aids discoverability in a form library. Active is the established term for nav/breadcrumb current-page state. Verdict: keep — HTML-attribute parity is the stronger convention here.

N3 · single-letter icons.X Low

icons/icon_names.go:20 — X Name = "x" (the close icon). Single-letter identifiers are normally a smell, but this is the canonical Heroicons name for the close/X glyph. A Close alias (same value) was already added as the preferred discoverable name. Verdict: already mitigated — prefer icons.Close in new code; X kept for parity with Heroicons.
