The skill's 12 problems are calibrated for domain/business models (UserID branded types, Order lifecycles, foreign-key relationships). The one design decision worth discussing

D1 · Props have no Validate() error — render-time fallback instead

Tracked as TODO_LIST #33 / #62 (deferred). The library normalizes bad input at render time (unknown enum → fallback value, negative progress → clamped to 0) rather than rejecting it upfront. This is the correct tradeoff for a UI library: a component must never crash a page render because a consumer passed an unexpected enum value. Crashing is worse than rendering a fallback style.

Recommendation: keep render-time fallback for presentational props. Add Validate() only where invalid config produces a broken page that the user can't recover from — e.g. errorpage.ErrorPageProps rendering a malformed error page. Do not blanket-add Validate() to all 74 structs; that is the TODO's "design decision needed" and the answer is "mostly no."

What the model gets right

Impossible states are largely unrepresentable

Closed-set enums (Family, OverlayKind, TableCellPadding, SortDirection) mean a prop can only hold a documented value or the fallback — never an arbitrary string. HxSwap htmx.SwapStyle is a typed enum, not a raw string. DrawerSide constrains side to left/right.

Composition without inheritance

Every props struct embeds utils.BaseProps and auto-satisfies ComponentProps via promoted pointer-receiver methods (GetBaseProps/SetBaseProps). This is struct embedding as composition — no AbstractProps/BasePropsImpl hierarchy. New components get the full BaseProps contract (Class/Attrs/ID/AriaLabel/Nonce propagation) for free.

Generics where they earn their keep

utils.Lookup[K comparable, V any] eliminated the per-enum if ok boilerplate across all 41 lookup maps. lookupFeedbackStyle[T]() shares the feedback style resolution across Alert/Toast/ErrorAlert. Ternary[T] for templ conditional rendering. Generics are used surgically, not fashionably.

The errorpage model is genuinely well-typed

Family (5 typed values) → familyStyleMap → distinct visual treatment. StatusCode int overrides family-derived defaults explicitly. CauseChain []CauseItem, Context []ContextPair. FromError() bridges go-error-family with errors.AsType. This is the strongest part of the model.

Conclusion

No redesign is warranted. The Props model is already a good fit for a server-rendered UI library: typed enums, composition-via-embedding, surgical generics, and crash-safe render-time fallback. The single open design question (D1) has a clear answer: keep fallback for rendering, add Validate() only for the errorpage handler. Update TODO_LIST #62 to reflect that decision rather than treating it as a 74-struct migration.
