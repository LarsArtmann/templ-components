Findings

F1 · scripts/release.sh — duplicated commit-body summary (Medium)

Medium · scripts/release.sh:137

RELEASE_BODY opens with ${RELEASE_SUMMARY} and the commit subject (line 145) repeats release: … — ${RELEASE_SUMMARY}, so every script-cut release gets a verbatim-duplicated first body line. Identified by the v0.18.0 post-mortem; still unfixed at HEAD.

Fix: drop the leading ${RELEASE_SUMMARY} from RELEASE_BODY.

F2 · scripts/release.sh — hardcoded wrong model attribution (Medium)

Medium · scripts/release.sh:143

The footer hardcodes Assisted-by: Crush:MiniMax-M3. The v0.18.0 tag now falsely claims MiniMax-M3 cut the release (actual: glm-5.2). v0.17.0 was attributed correctly by hand.

Fix: read the model from crush_info / a CRUSH_MODEL env var, or omit the line.

F3 · scripts/release.sh — hostile stdin prompt, no CHANGELOG fallback (Low)

Low · scripts/release.sh:102-109

Empty stdin (the automation path) produces an anemic commit body that violates the 3-paragraph AGENTS.md release spec. The script never derives notes from the CHANGELOG [Unreleased] block.

Fix: add --notes-file or auto-derive the body from CHANGELOG when stdin is empty.

F4 · GOEXPERIMENT=jsonv2 build requirement (Noted, documented)

Noted · errorpage imports encoding/json/v2

Consumers who go get without GOEXPERIMENT=jsonv2 get a compile error in errorpage. This is documented in ADR 0013 and the pre-commit hook / flake apps set the flag automatically. Not a defect, but real adoption friction to call out in the README install steps — and to remove once Go 1.27 promotes json v2 to stable.

F5 · Empty local cmd/ directory (Noted, untracked)

Noted · cmd/

An empty cmd/ directory exists locally but is not tracked by git (no files). It does not appear in clones. Related to deferred TODO #42 (shadcn-style CLI). Harmless; can be removed locally or filled when the CLI lands.

Strong patterns to preserve

Type safety is taken seriously

37 closed-set enums, each with IsValid() + a table-driven test; lookup maps use typed enum keys (map[Family]familyVisualStyle, not map[string]) and route through utils.Lookup. There is zero interface{} in production code — generics (Ternary[T], Lookup[K,V]) are used where dynamic behavior is needed.

Doc comments explain WHY, and warn about traps

e.g. utils.Ternary documents eager evaluation and points to the templ if/else alternative; overlayDialogJS documents why e.target === dialog detects backdrop clicks. This is the kind of commenting that survives staff turnover.

Drift-guard tests block regression at CI

Version ↔ CHANGELOG ↔ FEATURES counts are enforced by TestDocsCountDrift, TestVersionMatchesChangelog, dark-mode + motion-reduce compliance tests, and the CSP nonce integration test. This is why the docs stayed fresh.

Native web platform over JS

Modal/Drawer on <dialog>, Accordion on <details>, Carousel on scroll-snap, Textarea auto-grow on field-sizing, Select on appearance: base-select. ~200 lines of focus-trap JS eliminated in v0.18.0. This is the right direction and should continue.

No TODOs were added to source files during this review — there is nothing worth deferring. The release-script defects belong in scripts/release.sh directly (fixed, not annotated).
