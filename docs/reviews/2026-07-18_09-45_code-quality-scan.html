Findings

# Severity Location Issue Suggested fix
1 Medium scripts/release.sh:137 Duplicated release-summary line. RELEASE_BODY starts with ${RELEASE_SUMMARY}, and the commit subject (release: … — ${RELEASE_SUMMARY}, line 145) repeats it. Every script-cut release gets a verbatim-duplicated first body line. Drop the leading ${RELEASE_SUMMARY} from RELEASE_BODY; keep only the "rolls up changes since…" paragraph and the footers.
2 Medium scripts/release.sh:143 Hardcoded wrong attribution Assisted-by: Crush:MiniMax-M3. The v0.18.0 tag now falsely claims MiniMax-M3 cut the release; the actual model was glm-5.2. v0.17.0 was attributed correctly by hand. Read the model from crush_info (or a CRUSH_MODEL env var), or omit the line and let the operator add it. Never hardcode a model name.
3 Low scripts/release.sh:102-109 Hostile interactive stdin prompt for release notes. Feeding empty stdin (the automation-friendly path) produces an anemic commit body that violates the 3-paragraph AGENTS.md release spec. The script never derives notes from the CHANGELOG [Unreleased] section. Add a --notes-file flag, or auto-derive the commit body from the CHANGELOG [Unreleased] block when stdin is empty.
