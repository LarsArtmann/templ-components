# Migration Guide: v0.1 → v0.2

This guide covers breaking API changes introduced in the v0.2 release.

## Avatar: Status Field

**Before (v0.1):**

```go
Avatar(AvatarProps{
    Src:     "/user.jpg",
    Online:  true,   // bool
    Offline: false,  // bool — could set both to true!
})
```

**After (v0.2):**

```go
Avatar(AvatarProps{
    Src:    "/user.jpg",
    Status: AvatarStatusOnline,   // or AvatarStatusOffline, AvatarStatusNone
})
```

The `AvatarStatus` enum makes invalid states (both online and offline) unrepresentable.

## StatCard: Struct-Based API

**Before (v0.1):**

```go
StatCard("1,234", "Total Users", "+12%", true)
```

**After (v0.2):**

```go
StatCard(StatCardProps{
    Value:  "1,234",
    Label:  "Total Users",
    Change: "+12%",
    Trend:  TrendUp,    // or TrendDown, TrendNone
})
```

The positional API was replaced with a struct for clarity and extensibility. `TrendDirection` replaces the boolean `positive` parameter.

## PageProps: HTMX SRI

**Before (v0.1):**

```go
PageProps{
    HTMXSRI: "sha384-abc123...",  // SRI hash string
}
```

**After (v0.2):**

```go
PageProps{
    HTMXUseSRI: true,  // SRI hashes are computed automatically from version
}
```

SRI hashes are now auto-computed from `HTMXVersion`. Set `HTMXUseSRI: true` to enable.

## Security Headers Default

**Before (v0.1):** Security headers (`X-Content-Type-Options`, `Referrer-Policy`) were not included by default.

**After (v0.2):** `SecurityHeaders` defaults to `true` in `DefaultPageProps()`. To opt out:

```go
props := DefaultPageProps()
props.SecurityHeaders = false
```

## Form Error Attributes

All form components (Input, Select, Textarea, Checkbox) now use a shared `ErrorAttrs()` helper internally. The rendered HTML output is unchanged, but if you were testing internal attribute blocks, note that `aria-invalid` and `aria-describedby` are now set via `templ.Attributes` spreading.

## Icon Fields: Type-Safe icons.Name

**Before (v0.1):**

```go
DropdownItem{Text: "Edit", Icon: "pencil"}
EmptyStateProps{Icon: "folder"}
```

**After (v0.2):**

```go
DropdownItem{Text: "Edit", Icon: icons.Pencil}
EmptyStateProps{Icon: icons.Folder}
```

Icon fields changed from `string` to `icons.Name` for compile-time type safety. This prevents typos, enables IDE autocompletion, and makes invalid icon names unrepresentable. Import the `icons` package: `"github.com/larsartmann/templ-components/icons"`.

## Badge Size Constants

**Before (v0.1):**

```go
BadgeProps{Size: BadgeSizeSm}  // lowercase suffix
```

**After (v0.2):**

```go
BadgeProps{Size: BadgeSizeSM}  // uppercase suffix
```

All size constants now use uppercase suffixes for consistency: `BadgeSizeSM`, `BadgeSizeMD`, `BadgeSizeLG`.

## Spinner Size Constants

**Before (v0.1):**

```go
Spinner(feedback.SpinnerMedium, "text-blue-600")  // word suffix
```

**After (v0.2):**

```go
Spinner(feedback.SpinnerMD, "text-blue-600")  // uppercase suffix
```

All spinner size constants renamed: `SpinnerSmall`→`SpinnerSM`, `SpinnerMedium`→`SpinnerMD`, `SpinnerLarge`→`SpinnerLG`.

## Tabs Variant Naming

**Before (v0.1):**

```go
TabsProps{TabStyle: TabsStylePills}
```

**After (v0.2):**

```go
TabsProps{Variant: TabsPills}
```

Type renamed `TabsStyle`→`TabsVariant`, field renamed `TabStyle`→`Variant`, constants renamed `TabsStyleDefault`→`TabsDefault`, `TabsStylePills`→`TabsPills`.

## SimpleCard: Struct-Based API

**Before (v0.1):**

```go
SimpleCard()
```

**After (v0.2):**

```go
SimpleCard(SimpleCardProps{
    Padding: CardPaddingSM,
})
```

`SimpleCard()` was a zero-argument component. Now it takes `SimpleCardProps` (embedding `BaseProps`) for convention consistency with all other components. Use `DefaultSimpleCardProps()` for sensible defaults.

## TrendNone: Non-Empty Sentinel

**Before (v0.1):**

```go
TrendNone TrendDirection = ""  // empty string, indistinguishable from unset
```

**After (v0.2):**

```go
TrendNone TrendDirection = "none"  // explicit non-empty sentinel
```

If you were comparing `Trend == ""` to check for no trend, change to `Trend == TrendNone` or `Trend == "none"`.
