# Migrating to v1.0

This document lists the breaking changes in v1.0.0 and how to update your code.

## Removed: deprecated aliases

The following symbols were deprecated in v0.x and are now removed:

| Removed                           | Replace with                                     |
| --------------------------------- | ------------------------------------------------ |
| `display.ModalSizeFull`           | `display.ModalSize2XL` (same `max-w-4xl` width)  |
| `display.DrawerFull`              | `display.DrawerSize2XL` (same `max-w-2xl` width) |
| `errorpage.FamilyFromErrorFamily` | `errorpage.FromErrorFamily`                      |
| `forms.FormProps.Inline bool`     | `forms.FormProps.Layout: forms.FormLayoutInline` |

### Example migration

```go
// Before (v0.22)
display.Modal(display.ModalProps{Size: display.ModalSizeFull})
display.Drawer(display.DrawerProps{Size: display.DrawerFull})
forms.Form(forms.FormProps{Inline: true})

// After (v1.0)
display.Modal(display.ModalProps{Size: display.ModalSize2XL})
display.Drawer(display.DrawerProps{Size: display.DrawerSize2XL})
forms.Form(forms.FormProps{Layout: forms.FormLayoutInline})
```

## Added: `Validate() error` on `errorpage.ErrorPageProps`

`ErrorPageProps.Validate()` returns an error when:

- `Family` is not one of the five defined constants
- `StatusCode` is set but outside `[400, 599]`
- The page has no `Title`, no `Message`, and no `CauseChain` (would render empty)

Validate is **opt-in** — the renderer does not call it automatically. Call it
explicitly in handlers where you want to catch programmer errors:

```go
if err := props.Validate(); err != nil {
    return fmt.Errorf("invalid error page props: %w", err)
}
```

## Defaults NOT flipped (deferred to v2.0)

The original plan called for flipping HTMX self-host and semantic tokens to
defaults at v1.0. **This flip is deferred to v2.0.** Reason: both shipped
opt-in in v0.22.0 and a single minor cycle is not enough deprecation time.

What this means:

- `PageProps.HTMXSrc` remains opt-in (CDN is still the default).
- `templates/app.css` does import `templ-components-theme.css` by default —
  this is non-breaking because the theme file only adds aliases, it does not
  override any visible color.

When v2.0 ships, the HTMX self-host default will flip. Consumers upgrading
should vendor `htmx.min.js` at their chosen path (set via `HTMXSrc`).

## CI: docs-health drift guard

CI now runs `go test ./utils/... -run TestDocsCountDrift` on every push and PR.
This test fails when the component count, generated file count, or enum count
in `FEATURES.md`, `AGENTS.md`, `skill/SKILL.md`, or `website/src/data/sections.ts`
drifts from the actual count. Update docs counts in the same commit that adds
or removes a component.
