## Executive Summary

Migrated templ-components from a custom generic `Ptr[T]()` helper to Go 1.26's built-in `new(v)` keyword that accepts variables. Removed the now-redundant `Ptr[T any](v T) *T` from `utils/utils.go` (including its `//go:fix inline` directive) and updated the single call site in `utils/utils_test.go:47`. All tests pass (146 PASS, 0 FAIL). Build clean, lint clean. Coverage unchanged at ~68–69%.

**Rationale:** Go 1.26 extended `new()` to accept any expression (not just types), making `func Ptr[T any](v T) *T { return new(v) }` obsolete. The built-in is clearer, requires no import or helper, and eliminates maintenance of the `//go:fix inline` directive.

**Note:** `Deref` and `DerefOr` remain in `utils/utils.go` — these are still useful as they handle nil-pointer safety, which `new()` does not replace.
