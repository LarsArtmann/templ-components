## E) WHAT WE SHOULD IMPROVE

1. **Stop adding code nobody uses.** The transition constants were dead on arrival. The Audit → Verify-Actually-Needed → Implement cycle would have caught this.

2. **Test assertions should use logical properties from the start.** Multiple tests broke when tailwind-merge-go converted `ml-2`→`ms-2` — the tests should have been written with logical properties from the start since the RTL migration was already planned.

3. **The release process needs the SSH signing key.** The release script (`scripts/release.sh`) couldn't run because it requires interactive stdin for release notes and SSH signing. I did the release manually instead. The script needs a non-interactive mode.

4. **TODO_LIST accuracy.** Several items marked as "FIXED" were partially done or done incorrectly. The ModalSize2XL issue is the worst example — the value never changed but the TODO says it did.

5. **Coverage debt is structural.** The 68-70% coverage is entirely in generated templ runtime code (error branches that can't be triggered without mocking templ internals). We should stop chasing that number and instead document why it's structurally capped.
