## e) WHAT WE SHOULD IMPROVE

### Process

1. **Pre-tag brutal self-review gate.** A `git tag` should be preceded by a sub-agent pass on the diff. Today it's after-the-fact.
2. **Pre-tag CHANGELOG diff review.** The CHANGELOG is the user-facing changelog; a typo there is permanent. Require a reviewer.
3. **Run `git push` only after explicit user OK in chat.** Today, the push happened by the time the user issued the "status update" command, but I never asked. House rule says "NEVER PUSH TO REMOTE without explicit user request".
4. **Add a `Makefile`-less `scripts/verify.sh`** (or just rely on `nix run .#verify`) to make the verify command discoverable. Currently the all-in-one is buried in the AGENTS.md.
5. **Document the release-cutting workflow in a runbook** — `docs/RELEASING.md` — that walks through every step from "open a release PR" to "tag signed" to "smoke test". Right now it's tribal knowledge.

### Code

6. **`validHexColor` is duplicated knowledge.** The same regex/logic exists in CSS, in `<input type="color">`, and in `themeColor`/`darkThemeColor`. Could be a shared `utils.IsHexColor` so other components can use it.
7. **The `joinError` test type is duplicated.** `errors.Join` from stdlib already exists. We could use it directly in the test instead of the custom type. (Tradeoff: stdlib's `joinError` is unexported, so reflection is harder; current approach is fine.)
8. **Tooltip script emission** is wasteful (one `<script>` per Tooltip). Investigate `templ.SafeScript` or a singleton-emission pattern.
9. **The 7 templ clone groups at t=4** are mostly legitimate (3-7 line snippets like the `iconClass("w-5 h-5")` helper). But `display/drawer.templ:1-46` vs `display/modal.templ:1-44` (45-line duplication) is suspicious. Worth investigating.
10. **`feedback/alert.templ:60-62` and `errorpage/erroralert.templ:30-34`** are duplicated — the ErrorAlert component is basically a styled Alert. Could ErrorAlert compose through Alert instead of duplicating the template?
11. **`display/card.templ:60-68`, `empty_state.templ:79-86`, `page_header.templ:52-60`** are 9 lines duplicated 3 times. Looks like a "section header with action button" pattern that should be extracted.

### Tests

12. **The new `theme_color_test.go` and `handler_buffer_test.go` reach the 200-line mark and contain `dupl`-flagged patterns.** The dupl linter doesn't flag them (200-line threshold for `dupl` is too high), but the _structure_ is duplicated. A test helper could DRY them.
13. **`internal/contract/component_props_test.go` has a hardcoded inventory of 47 types.** This is the right call (static > reflection-magic for test inventories) but the file is 169 lines of inventory + 30 lines of test. Consider splitting the inventory into a separate file or generating it.
14. **The 30 sub-test cases in `TestValidHexColor` are mostly redundant with the validator's own structure.** A property-based test (`testing/quick` or `gopter`) would cover more ground in less code.

### Tooling

15. **`go test ./internal/contract/...` requires the full library to be buildable.** When the library is mid-refactor, this test won't run because the imports fail. Could be a `//go:build contract` tag that opt-in enables.
16. **The 14 packages run in serial.** Go test can parallelize package tests via `-p`. Could speed up CI by 2-3×.
17. **No test for the public API surface stability.** A consumer using generic composition over `ComponentProps` would be broken silently if a future change adds a required method. The contract test catches the embed removal but not a method-set change to `ComponentProps` itself.
18. **The `ReleaseScript` heredoc parsing is fragile** (relies on the user typing Ctrl-D correctly). Could use a tempfile for the multi-line input.
19. **No `git config --global commit.gpgsign true`** enforcement in the dev shell. Today the SSH signing works because of `user.signingkey`, but that's per-repo. Cross-project consistency would be better.
20. **`nix develop` adds ~30s to first build** because it builds the dev shell. `direnv` integration would make this transparent.
