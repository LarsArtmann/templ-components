Current state

The project is already on the standard flake foundation — flake-parts, systems, nixos-unstable, mkShellNoCC, and five apps (build/test/lint/verify/coverage). nix flake check --no-build passes. There is no justfile or Makefile. The migration is therefore a refinement, not a from-scratch migration.

Notably, the website/flake.nix already uses treefmt-nix — so the pattern is proven in this repo. The root flake just hasn't adopted it yet.

Proposed flake.nix additions

Add treefmt-nix as an input and a treefmt module + checks block inside perSystem. Below is the delta (not a full rewrite — the existing apps/devShell stay).

1. New inputs

# add to inputs:
treefmt-nix = {
  url = "github:numtide/treefmt-nix";
  inputs.nixpkgs.follows = "nixpkgs";
};

2. Module import + treefmt config + checks (inside mkFlake)

flake-parts.lib.mkFlake { inherit inputs; } {
  systems = import inputs.systems;
  imports = [ inputs.treefmt-nix.flakeModule ];  # NEW

  perSystem = { config, pkgs, ... }: {
    # ... existing devShells / apps unchanged ...

    # NEW: replaces the bare `formatter = pkgs.nixfmt;`
    treefmt = {
      projectRootFile = "go.mod";
      programs = {
        gofumpt.enable = true;
        goimports.enable = true;
        nixfmt.enable = true;
        templ.enable = true;
      };
    };

    # NEW: make `nix flake check` meaningful
    checks = {
      format = config.treefmt.build.check self;
      build = pkgs.runCommand "tc-build-check" {
        nativeBuildInputs = [ pkgs.go_1_26 pkgs.templ ];
        GOWORK = "off"; GOEXPERIMENT = "jsonv2";
      } ''
        cp -r ''${src}/* .
        find . -name '*_templ.go' -delete
        templ generate ./...
        go build ./...
        touch $out
      '';
    };
  };
};
Note on the build check

The src for the build check should use lib.fileset to avoid copying website/, docs/, .git. The skeleton above is illustrative; the real implementation should mirror the fileset discipline from the skill's Go template.

Why NOT git-hooks.nix

BuildFlow already owns pre-commit

.git/hooks/pre-commit is a BuildFlow-generated hook (buildflow precommit install) running BuildFlow in pre-commit mode with a 60s budget. git-hooks.nix would install a second, competing pre-commit system. Pick one. BuildFlow is already configured and working, with library-specific checks (templ generate, the GOEXPERIMENT=jsonv2 guard, dupl_threshold). Keep BuildFlow; do not add git-hooks.nix.

Post-migration checklist

  • nix flake check --no-build passes (already true)
  • flake-parts used (not flake-utils)
  • systems input used (not hardcoded list)
  • mkShellNoCC for shells
  • just not present in devShell
  • treefmt-nix imported with nixfmt + gofumpt + goimports + templ
  • checks.format and checks.build defined
  • nix fmt formats nix + go + templ files
  • nix build not required (library; demo binary optional)
  • Verify BuildFlow pre-commit still runs after treefmt is added (no conflict — treefmt is nix fmt, BuildFlow is git-hook)

Recommendation

Low-priority refinement. The flake is already valid and the dev workflow works. The two additions (treefmt-nix + checks) are pure improvements that bring the root flake to the same standard the website flake already meets, and make nix flake check + nix fmt meaningful for CI. Estimated effort: ~30 minutes. No urgency — schedule alongside the next flake touch.
